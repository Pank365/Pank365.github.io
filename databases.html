Artifact Overview

This artifact is a MongoDB-based animal shelter management application originally created in CS-340. It includes a Python CRUD module used to connect to a MongoDB database and a Dash dashboard used to display and interact with stored records. This made it a strong fit for the database category because it demonstrates data storage, retrieval, update, delete behavior, and integration between a backend database and a user-facing interface.

  • Added better error handling and helper methods such as count() and close().

Enhancement Narrative

The artifact I selected for this enhancement is a database-driven animal shelter management application originally created in CS-340: Client/Server Development. The project was designed to connect a Python application to a MongoDB database and provide a way to create, read, update, and delete animal shelter records. The artifact also included a Dash dashboard that displayed the database records in a more user-friendly and interactive format.

I selected this artifact for my ePortfolio because it demonstrates my ability to design and implement a software solution that integrates a database with an interactive application interface. It highlights both backend and frontend development skills, as well as my understanding of database connectivity, query handling, and modular software design.

To enhance the artifact, I improved the CRUD module by making it more flexible, secure, and maintainable. I expanded the read() method to support optional query projection, sorting, and limiting. I also improved validation in the create() method, added safer control options to the update() and delete() methods, and strengthened error handling throughout the module.

Through this enhancement, I demonstrated stronger database design practices and a better understanding of how data management, application structure, and usability work together in a practical software solution.
